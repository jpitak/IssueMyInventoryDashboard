Design Upgrade ที่เพิ่มให้

สิ่งที่ถูกเสริมเข้าไปจากมุมมอง dashboard design และ usage จริง
1) Executive-first layout
เปิดมาเห็น KPI สำคัญก่อน ได้แก่ Total, Open, High Priority, Overdue และช่วงวันที่ของข้อมูล
2) Multi-page logic
แยกเป็น Overview, Operations, Registry และ Recommendations เพื่อให้ใช้งานได้ทั้งผู้บริหารและทีมปฏิบัติการ
3) Actionable registry
เพิ่มหน้า filter/search สำหรับใช้แทนการไล่ดู Google Sheet โดยตรง
4) Data quality visibility
ชี้ให้เห็นช่องโหว่ด้านข้อมูล เช่น ไม่ระบุวัน, ไม่ระบุ assignee, ไม่ระบุ status/priority

Recommendation เพื่อให้ dashboard ใช้งานได้ดีกว่าเดิม

ข้อเสนอเชิงโครงสร้างข้อมูลและการบริหาร issue
1) กำหนดมาตรฐานฟิลด์บังคับ (Data Validation)
ป้องกันการปล่อยให้ช่องว่าง (Blank) ในระบบ โดยเฉพาะในส่วนฟิลด์ วันที่แจ้ง, สถานะ และผู้รับผิดชอบหลัก
2) ตั้งโครงสร้างสถานะให้เป็นระบบ
จัดลำดับสิทธิ์การย้ายสเตตัส เช่น New ➡️ In Process ➡️ SA Testing ➡️ Complete เพื่อลดการระบุเป็น Unspecified
